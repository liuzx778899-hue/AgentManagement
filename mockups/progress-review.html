设计成熟度 Review

维度 状态 判断
产品定位 清楚 个人本地优先的 AI 软件项目经理工作台，后续预留团队模式。
信息架构 基本清楚 Project、Task、Role、Workflow、Gate、Memory、Capability Pack、Runner Log 已成型。
视觉方向 可用 深色、高密度、工程反馈优先，适合开发者工作台。
第一版主路径 待收口 项目接入、新建任务、启动 Agent、Manual Gate、合并/保存记忆还需要串成完整闭环。
实施计划 草案 已有 MVP plan，但应在 V1 flow 锁定后微调。

已经确定

  • 工作台采用左侧导航、中间任务执行、右侧工程反馈的三栏结构。
  • Workflow 是独立资源，不只是项目设置里的字段。
  • Role Library 和 Launcher/Profile 需要拆开，角色不能写死成枚举。
  • Manual Gate 是核心交互，高风险操作必须人工确认。
  • Codex 集成以本地入口或弱集成为主，不假设 ChatGPT 订阅额度能被第三方 API 消耗。

还没锁定

  • 用户创建任务的完整流程。
  • Workflow Builder 的步骤编辑、版本升级和项目覆盖交互。
  • Manual Gate 中展示哪些证据和风险。
  • 记忆建议是否能在第一版保存、编辑、引用。
  • 普通用户配置和高级 Launcher 配置的边界。

建议先锁定的 V1 主路径

01

接入项目

导入仓库，配置测试、构建、预览命令和项目记忆。

02

新建任务

输入目标，选择项目和 Workflow，确认角色与能力包。

03

启动 Agent

创建 worktree，执行步骤，记录日志、产物和权限。

04

Manual Gate

基于 diff、测试、截图、日志和风险做继续或改派决策。

05

收尾

合并、保存记忆、生成总结，并进入下一任务。
