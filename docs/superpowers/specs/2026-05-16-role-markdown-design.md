# 角色 Markdown 设计方案

日期：2026-05-16
状态：设计方案

## 目标

为每个 AgentRole 增加 Markdown 行为定义（类似 CLAUDE.md 的角色版），AI 自动生成基础框架 + 人手动补充细节。在工作流画布和项目工作区两个位置均可编辑和查看。

## 数据模型

```typescript
// AgentRole 新增字段
interface AgentRole {
  // ... 现有字段 (id, projectId, name, description, isBuiltIn, defaultCapabilities)
  roleMarkdown: string;  // 角色行为 Prompt（Markdown）
}

// Project 新增字段
interface Project {
  // ... 现有字段
  roleOverrides?: Record<string, string>;  // roleId → 项目级覆盖的 Markdown
}
```

**优先级规则：** 项目级 roleOverrides[roleId] 有值 → 用项目级；否则 → 用 AgentRole.roleMarkdown。

## 自动生成规则

新建角色时，根据 `name + description + defaultCapabilities` 自动生成初始 MD：

```markdown
# {角色名称}

**描述：** {description}

## 核心职责
（根据 description 生成）

## 可用能力
- {capability1}
- {capability2}

## 行为规则
- 在进入下一步之前，将工作结果提交给人工决策
- 将关键结论写入项目记忆

## 输出格式
- 使用中文回复
- 代码块标注语言类型
```

## 组件拆分

| 组件 | 文件 | 职责 |
|------|------|------|
| `RoleMdEditor.tsx` | NEW | Markdown 编辑器覆盖层：textarea + 预览 + 重新生成 + 保存 |
| `RoleMdViewer.tsx` | NEW | Markdown 只读查看器：渲染 MD + 切换到编辑 + 显示来源 |

**修改已有组件：**

| 组件 | 改动 |
|------|------|
| `WorkflowBuilder.tsx` | 角色卡片加 [📝] 按钮，点击打开 RoleMdEditor |
| `ProjectWorkspace.tsx` | Agent 信息栏加 [📄 查看角色 MD] 按钮，点击打开 RoleMdViewer |
| `NewProjectWizard.tsx` | 新建角色时自动生成 roleMarkdown |

## 交互要点

1. **AI 自动生成** — 新建角色时根据 name/description/capabilities 生成初始 MD
2. **重新生成** — 编辑器中有"重新生成"按钮，覆盖当前内容
3. **项目级覆盖** — 工作区中编辑保存到 Project.roleOverrides
4. **来源标识** — 查看时标注"当前使用：默认 / 项目级覆盖"
5. **重置** — 项目级可重置为角色默认 MD
