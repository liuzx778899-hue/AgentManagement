# 项目工作区 V2 设计方案

日期：2026-05-16
状态：设计方案
效果参考：`mockups/project-workspace-v2.html`

## 目标

重构 ProjectWorkspace 页面，以**项目进度看板**为主体，将运行状态指标收缩到工作流执行进度面板内，项目记忆独立为顶部按钮+弹窗。

## 与 V1 的关键差异

| 区域 | V1 | V2 |
|------|----|----|
| **主体** | stats 行 + task card + stepper + log + 文档 | **进度总览卡片**（环比 + 进度条 + 当前阶段）+ TODO/Stepper/Log/Docs 3 层网格 |
| **运行状态** | 4 卡片占据左侧顶部 | 收缩到「当前任务」面板 header 内（运行中 · 等待决策 · 已完成 · 记忆数 inline 小数字） |
| **项目记忆** | 散落在左侧 stats 中 | 顶部栏独立 `[项目记忆]` 紫色高亮按钮 → 弹出管理面板 |
| **文档面板** | 独立 section | 与 Agent 运行日志并列底部两栏 |
| **无** | — | 点击进度圆环 → 弹出「总体进度仪表板」overlay |

## 组件修改

### ProjectWorkspace.tsx（重写左侧主体）

**进度总览卡片（ProgressCard）**
- 左侧：环形进度圆环（可点击，cursor:pointer）
  - 点击 → 弹出「总体进度仪表板」overlay
  - hover 亮度提升 + 按压缩放
- 右侧：4 条进度条（已完成/运行中/待决策/记忆）
- 底部：当前阶段横幅（橙色脉冲圆点 + 步骤名+状态描述）

**2 列面板网格**
- 左列：待办清单（TODO List）— 步骤 checklist，✓/⬜ 状态
- 右列：当前任务 + 工作流 Stepper（紧凑步骤卡片 + 连接线）

**底部 2 列**
- 左列：协同文件 & 设计文档（分类标签 + 文件列表）
- 右列：Agent 运行日志（等宽字体 + 时间 + 状态图标）

### 总体进度仪表板 Overlay（ProgressDashboard）

点击环形进度圆环 → 弹出 overlay，内容复用 `mockups/progress-dashboard.html` 风格：
- Phase 1/2/3 三阶段路线图
- 关键指标（组件数/测试/构建错误/代码行数）
- Phase 1 子阶段明细（P0-P6 进度条）
- Phase 2 判定条件
- 执行路径（Step 1→修复构建 → Step 2→P5收尾 → Step 3→P6完善 → Phase 2）

### 项目记忆面板（MemoryPanel）

顶部栏 `[项目记忆]` 按钮 → 弹出 overlay：
- 记忆列表（项目记忆/角色记忆/任务记忆 tabs）
- 手动输入 textarea + 保存按钮
- 已有记忆展示（标题/来源/引用次数/自动或手动标签）

### App.tsx / Topbar 改动

- 新增 `[项目记忆]` 按钮状态 → memoryPanel open/close
- 新增 progressDashboard overlay 状态
