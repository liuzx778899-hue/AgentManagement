# 全局 AI 助手设计方案

日期：2026-05-16
状态：设计方案

## 目标

设计一个在所有页面可用的全局 AI 工程助手，能理解当前页面上下文，读取和写入系统数据，引导用户完成配置工作流，生成的内容可直接应用到系统中。

## 核心能力

1. **页面感知** — 知道用户当前在哪个页面（工作台/项目管理/工作流画布/项目工作区/设置等），主动提供相关帮助
2. **系统数据读写** — 能读取当前页面的数据（工作流模板、角色列表、项目配置等），生成并写入修改
3. **引导式操作** — 主动发现缺失配置（如"角色还没设定 Prompt"），引导用户完成配置步骤
4. **一键应用** — AI 生成的内容（工作流模板、角色 MD、项目配置）可直接应用到系统数据

## 页面感知 + 智能建议

助手根据当前页面类型，提供不同的上下文能力：

| 页面 | 可读取数据 | 可执行操作 |
|------|-----------|-----------|
| **工作流画布** (`workflows`) | 当前模板、步骤列表、角色列表 | 生成/修改工作流模板、生成步骤规则 MD、优化流程设计 |
| **项目工作区** (`project-workspace`) | 项目信息、当前任务、工作流进度、Agent 状态 | 生成项目 MD、角色 MD 覆盖、分析执行状态 |
| **项目管理** (`project-management`) | 项目列表、任务列表、Gate 状态 | 生成项目配置、批量创建任务、分析项目健康度 |
| **设置中心** (`settings`) | 模型配置、IM 适配器、Git 认证 | 生成配置建议、检查配置完整性 |
| **工作台** (`workbench`) | 全局任务队列、运行状态 | 生成任务优先级建议、运行报告 |

## 引导式操作流程

助手会主动检查缺失配置并引导用户：

```
助手: 👋 我注意到你的"需求分析"步骤还没有配置角色 MD（行为 Prompt）。
     需要我帮你生成吗？
     
     [生成角色 MD] [暂时跳过]

用户点击 [生成角色 MD] →

助手: 好的，根据角色"产品经理"的职责和能力，我生成了以下 Prompt：
     
     [显示生成内容预览]
     
     [应用到角色默认] [仅当前项目覆盖] [修改后应用] [取消]

用户选择 →

助手: ✅ 已将角色 MD 保存到"产品经理"。你还需要配置其他角色的 MD 吗？
```

## 可执行动作类型

| 动作 | 触发词示例 | 效果 |
|------|-----------|------|
| 生成角色 MD | "帮我写产品经理的Prompt" | 生成 MD → 预览 → 保存到 AgentRole.roleMarkdown |
| 设计工作流 | "设计一个Bug修复流程" | 生成步骤列表 → 预览 → 创建 WorkflowTemplate |
| 优化流程 | "当前流程有什么问题" | 分析模板 → 给出优化建议 → 一键应用修改 |
| 生成项目 MD | "写这个项目的上下文" | 基于 Project 数据生成 MD → 保存到 projectMarkdown |
| 检查配置 | "检查配置是否完整" | 扫描角色/流程/项目 → 列出缺失项 → 逐个引导完成 |
| 分析状态 | "当前运行情况如何" | 读取 tasks/agentRuns/gates → 生成状态报告 |

## 技术实现（Phase 1）

Phase 1 不接真实 AI API，用内置规则 + fixture 模拟：

- **意图识别**：关键词匹配（"生成"/"优化"/"设计"/"检查"等）
- **内容生成**：基于模板 + 现有数据填充（类似 MD 的自动生成模板）
- **操作执行**：通过现有的 WorkbenchContext 方法写入数据
- **上下文感知**：通过 `view` 状态确定当前页面

Phase 2 再接入真实 LLM API 实现智能对话。

## 组件拆分

| 组件 | 文件 | 职责 |
|------|------|------|
| `AiAssistant.tsx` | NEW | 全局助手容器：气泡按钮 + 浮动面板 + dock 状态 + 页面感知 |
| `AiChatPanel.tsx` | NEW | 聊天面板：消息列表 + 输入框 + 动作按钮渲染 |
| `AiActions.tsx` | NEW | 可执行动作：生成MD/创建模板/检查配置等 |

## 集成方式

在 `App.tsx` 中全局渲染 `<AiAssistant>`，不依赖路由，始终存在：

```tsx
// App.tsx
<AiAssistant view={view} data={data} />
```

这样子助手在所有页面都可用，并知道当前 `view` 是什么。
