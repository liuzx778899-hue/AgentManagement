# Workflow Canvas Design V2

日期：2026-05-16
状态：设计方案（更新版）
关联：`docs/HANDOFF_NEXT_TASKS.md`、`mockups/workflow-canvas-design.html`

## 目标

将 WorkflowBuilder 改造为**两栏布局**：左侧边栏（工作流模板列表 + 项目角色池）+ 右侧画布（紧凑步骤卡片 + 插入按钮 + 出口端口 + 双击编辑模态框）。

## 约束

- 无第三方可视化库，纯 React + TypeScript + CSS
- 深色主题，与现有设计 token 一致
- lucide-react 图标，中文界面
- **双击节点打开模态框编辑**（非内联展开）
- Phase 1 前端 MVP，fixtures 数据驱动
- 设计参考：`mockups/workflow-canvas-design.html`

## 组件设计

### 1. Topbar（顶部栏）

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  [GitBranch] 工作流编排  | 定义角色 Agent，编排步骤流程   [版本管理] [保存工作流] │
└──────────────────────────────────────────────────────────────────────────────┘
```

- 左侧：图标 + 标题「工作流编排」+ subtitle「定义角色 Agent，编排步骤流程」
- 右侧：[版本管理] [保存工作流] 按钮

### 2. 左侧边栏（Sidebar）

**工作流模板列表：**
- Header：● 绿点 + 「工作流模板」+ 数量 badge
- 模板卡片：
  - 图标 + 名称 + meta（版本 · 步骤数）
  - hover 显示操作按钮：[复制] [删除]
  - active 状态：绿色边框 + 淡绿背景
- 底部：[+] 新建工作流模板（虚线按钮）

**项目角色池：**
- Header：● 紫点 + 「项目角色池」+ 数量 badge + [+ 新增] 按钮
- 角色项：
  - 头像（首两字）+ 名称 + 描述
  - 模型 badge（如 DeepSeek v4）
  - 已绑定步骤：✓ 步骤1, 步骤3

### 3. 右侧画布（Canvas）

**Info Bar：**
- 图标 + 工作流名称 + meta（版本 · 日期）
- 右侧统计：步骤数 + 人工决策数（橙色）

**Toolbar Hint：**
- 提示文字 + 图标

**步骤卡片（Step Card）：**
```
┌──────────────────────────────────────────────────────────────────┐
│ [≡]  │  01  │  需求分析                          │ [●] [●] │
│ ←拖拽│      │  产品经理 | DeepSeek v4 | 🔴人工决策 │ 成功 失败│
└──────────────────────────────────────────────────────────────────┘
```

- **左侧拖拽手柄**：hover 显示，用于排序
- **序号徽章**：01、02 等，蓝色背景
- **信息区**：名称 + 角色 badge + 模型 badge + Gate badge
- **右侧出口端口**：
  - 成功端口（绿色圆点）：hover 显示「→ 步骤2」或「→ 完成」
  - 失败端口（橙色圆点）：hover 显示「→ 停止」或「→ 重试」

**连接线（Edge）：**
- 绿色成功线 + 输出名称标签 + 勾选图标 + 箭头
- 失败分支线（retry/fallback）：从失败端口引出

### 4. 步骤间插入按钮

```
                    ⊕  ← hover 显示圆形加号按钮
```

### 5. 步骤编辑模态框

```
┌─────────────────────────────────────────────────────────────────┐
│ [Edit] 编辑步骤 01 · 需求分析                           [×] 关闭│
├─────────────────────────────────────────────────────────────────┤
│ 步骤名称   [需求分析____________]   执行角色  [产品经理 ▼]       │
│ 绑定模型   [DeepSeek v4-pro ▼]    Gate 模式 [人工决策 ▼]       │
│ 输入       [项目背景, 用户需求__]  输出      [需求规格摘要____] │
├─────────────────────────────────────────────────────────────────┤
│ [ChevronRight] 连线走向配置                                      │
│ ✓ 成功时前往              [步骤 2 · UI/UX 设计 ▼]              │
│ ✗ 失败时操作              [停止流程 ▼]                          │
├─────────────────────────────────────────────────────────────────┤
│ [🗑] 删除步骤                              [取消] [✓ 保存]      │
└─────────────────────────────────────────────────────────────────┘
```

## 组件拆分

| 文件 | 职责 |
|------|------|
| `WorkflowBuilder.tsx` | （重写）Topbar + 两栏布局容器 |
| `WorkflowCanvas.tsx` | 画布渲染：InfoBar + ToolbarHint + 步骤列表 + 插入按钮 |
| `WorkflowNode.tsx` | 紧凑卡片：拖拽手柄 + 序号 + 名称 + badges + 出口端口 |
| `WorkflowEdge.tsx` | 连接线：绿色成功线 + 输出标签 + 箭头 |
| `StepEditModal.tsx` | 模态框：编辑表单 + 路由配置 + 删除 |

## 交互要点

1. **模板切换** — 点击左侧模板卡片，画布重新渲染
2. **双击编辑** — 双击步骤卡片打开模态框
3. **出口端口 hover** — 显示去向标签（→ 步骤2, → 停止, → 重试）
4. **拖拽手柄** — hover 显示，用于排序（Phase 1 可选）
5. **插入按钮** — hover 显示 ⊕ 按钮，点击在步骤间插入新步骤
6. **模板操作** — hover 模板卡片显示复制/删除按钮
